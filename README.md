# LLM Semantic Detection — 轻量LLM常识语义增强的实时开放词汇检测/分割系统

面向 **AIC 竞赛** 的轻量级开放词汇检测与分割系统。利用 **Qwen2.5-0.5B** 的常识推理能力，无需图像即可为任意物体名称生成细粒度结构化语义描述，增强 **SegEarth-OV3** 开放词汇模型的检测/分割精度。

## 项目背景

传统开放词汇检测模型仅依赖简单的类别名词（如 "car"），语义信息不足，导致细粒度目标识别效果差。本方案提出 **"无图常识语义增强"** 范式——仅输入物体名称，LLM 自动生成形体、材质、空间关系等描述，显著提升检测精度。

## 三大核心创新

| # | 创新点 | 说明 |
|---|--------|------|
| H1 | **无图常识语义增强** | 仅输入物体名称，Qwen2.5-0.5B 生成细粒度结构化描述，解决简单名词语义不足痛点 |
| H2 | **低时延轻量流水线** | LLM 纯文本生成（不处理图像）+ vLLM 推理优化，端到端 <60ms |
| H3 | **分层非对称 LoRA 优化** | 浅层低秩保留通用特征，深层高秩适配任务，不破坏原生开放词汇能力 |

## 系统架构

```
输入层: 任意物体名词 → 自然语言属性查询
    ↓
轻量 LLM 语义增强 (Qwen2.5-0.5B + vLLM)
    ↓
语义缓存 (字典缓存，重复调用免推理)
    ↓
SegEarth-OV3 文本&视觉编码器
    ↓
分层非对称 LoRA 跨模态对齐
    ↓
自适应置信度阈值 + 改进 NMS
    ↓
检测框 + 语义分割掩码 双输出
```

## 技术栈

| 模块 | 选型 |
|------|------|
| 轻量 LLM | Qwen2.5-0.5B-Instruct (4bit 量化, ~0.8GB 显存) |
| 推理框架 | vLLM (PagedAttention, 2-3×加速) |
| 开放词汇基线 | SegEarth-OV3 (曹相湧团队) |
| 优化方法 | 分层非对称 LoRA (PEFT) |
| 硬件适配 | Apple M4 Air / 任意消费级 GPU |

## 性能指标 (M4 Air 预估)

| 模块 | 耗时 | 
|------|------|
| LLM 生成 (30 token) | 5-8 ms |
| SegEarth-OV3 推理 | 35-40 ms |
| 后处理 (NMS + 阈值) | 8-10 ms |
| **端到端** | **48-58 ms (17-21 FPS)** |
